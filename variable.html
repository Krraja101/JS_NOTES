Variables :

  • A named block of memory which is used to store a value is known as variables (containers used to store data).
  • NOTE : In js variables are not strictly typed, it is dynamically typed.
  • Therefore it is not necessary to specify type of data during variable declaration.
  • In a variable we can store any type of value.
  • It is mandatory to declare a variable a variable before using.
  • Syntax : var/let/const identifier;

    • EX :-

    var a;//declaration
    a=10;//initialization or assignment

  • NOTE : When a variable in java script js engine implicitely assign Undefined value to it.

    • EX :-

    var a;
    console.log(a); //Undefined

    Understanding execution in java script :

  • Everytime when js engine run as js code, it will first create a "Global execution context".
  • The global execution context has two phases :
    1. Variable phase
      • All the memory is allocated for the declarationin in top to bottom order & assigned by default value "Undefined" in variable area of global execution context.
    2. Funtional/Executional phase
      • All the instructions get executed in top to bottom order in execution area of global execution context.

    Variable hoisting :

  • JS allows a programmer to use a number (variable) before the declaration statement, this charcteristics is known as variable hoisting.
  • Variable hoisting is only supported by "var" where as let and const doesn't support variable hoisting.
